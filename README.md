#Collaborative Development using GitHub
#Project Overview :-
This is a group micro-project for the Open Source Technology course at Government Polytechnic, Pune. It demonstrates how to use Git and GitHub for team-based version control and web hosting.

#Core Commands :-
   1.git init: Create local repo.

   2.git add .: Stage all changes.

   3.git commit -m "msg": Save changes with a note.

   4.git push origin main: Upload to GitHub.

   5.git pull: Get latest updates from the team.

#Workflow :-
   1.Setup: Created a shared repository for the group.

   2.Commit: Members staged and committed individual sections.

   3.Merge: Used Pull Requests (PR) to combine work into the main file.

   4.Deploy: Hosted the final blog/guide via GitHub Pages.

#Conclusion :-
This project successfully shows that Git allows a group to work on the same files simultaneously without conflicts. By using SHA-1 hashing and the staging process, we maintained a clear history and a single, cohesive final product. This workflow is the standard for professional software development.
